## ts-forecasting

**한 줄 요약**: 금융 시계열 예측. 36,923개 시리즈 중 시험 데이터의 89%가 처음 보는 새 시리즈.
**핵심 난관**: 새 시리즈에 대해 아무 정보 없이 예측해야 함. 그런데 특정 시리즈 하나를 조금만 틀려도 점수가 0이 됨.

### 삽질 기록 (0점 3번의 교훈)

**1차 삽질: val이 시험을 반영 못 함 (sub_02)**
시험 데이터에는 직전값(lag)이 없는데 val에서는 있었음. val score 0.89에 속아서 제출 → 0점.
→ 교훈: val을 시험과 동일한 조건으로 만들어야 함.

**2차 삽질: 고가중치 시리즈 폭발 (sub_03)**
83EG83KQ라는 시리즈의 중요도가 13조(다른 시리즈의 수백만배). 이 시리즈 실제값은 0.000009인데 6.37로 예측 → 오차가 5.6×10¹⁴ → 점수 0.
→ 교훈: 제출 전에 고가중치 시리즈 예측값 반드시 확인. 자동 패치 시스템 구축.

**3차 삽질: 그룹 평균이 독 (sub_04)**
새 시리즈(89%)에 "이 그룹의 훈련 평균은 -0.67이다"를 feature로 줬더니 모델이 -0.67 근처를 예측. 시험의 새 시리즈 실제값이 0에 가까우면 → -0.67 예측은 0으로 찍는 것보다 나쁨 → 누적되면 점수 0.
→ 교훈: 훈련 데이터의 y_target 기반 통계(그룹 평균, 시리즈 평균)를 새 시리즈에 쓰면 안 됨. raw feature만 안전.

**핵심 발견: 대회 호스트가 "public 0.5+ 점수는 미래 데이터 사용(반칙) 의심"이라고 발언. 정직한 접근의 ceiling이 0.3~0.5 수준일 수 있음.**

## 전체 교훈

1. **val score를 맹신하면 안 됨** — val이 시험 상황을 반영하는지 항상 확인
2. **제출 전 예측값 분포 확인 필수** — 점수가 0이 나오는 건 모델 문제가 아니라 예측값 폭발/편향
3. **대회 마감일 먼저 확인** — "X days to go"가 제출 마감이 아닐 수 있음
4. **y_target 기반 통계를 새 시리즈에 쓰면 위험** — raw feature만 안전
5. **단순한 모델이 더 안정적** — sub_01 (raw feature만)이 유일하게 0점 아닌 제출

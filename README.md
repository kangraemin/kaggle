## ts-forecasting

**한 줄 요약**: 금융 시계열 예측 대회. 모델이 과거 패턴을 보고 미래값을 맞추는 문제.
**핵심 난관**: 시험 데이터의 89%가 훈련 때 한 번도 본 적 없는 새로운 시리즈.

### 실험 흐름

| Trial | 왜 시도했나 | 결과 | 다음엔 |
|-------|------------|------|--------|
| 001 기본 모델 | 아무것도 모르니 주어진 feature 그대로 LightGBM 돌려보기 | public 0.1499. 생각보다 괜찮음 | "직전값이 다음값을 강하게 예측한다(상관관계 0.86)"는 걸 발견 → 직전값 feature 추가 |
| 002~010 직전값 feature | 직전값이 강한 신호니까 추가하면 점수 오를 것 | val 0.89로 폭등. 그런데 **제출하니 0.0000** | 시험 데이터엔 직전값이 없었다. val에서는 정답지 보며 채점한 꼴. val이 틀린 방법이었음 |
| 012~014 직전값 제거 | val 방식을 바꾸기 전에 일단 직전값 없이 재시도 | val 0.30. 새 시리즈의 평균값도 시험에서 89% NaN | val 자체를 시험 상황처럼 바꿔야 함 |
| 015~021 새 평가 방식 | 훈련 시리즈 일부를 숨기고 "처음 보는 시리즈" 상황 재현 | val 0.64. 그런데 **제출하니 또 0.0000** | 특정 시리즈(83EG83KQ)가 중요도(weight) 13조짜리인데 예측값을 크게 틀림 → 점수 공식 분자가 천문학적으로 커져서 0점 |
| 031~032 자동 안전장치 | 중요도 높은 시리즈 예측이 크게 벗어나면 자동으로 안전값으로 교체 | 이후 제출에서 0점 재발 없음 | 학습 데이터의 마지막 관측값을 feature로 활용해보기 |
| 033 마지막 관측값 | 알고 있는 시리즈(10.9%)에 한해 "훈련 마지막에 이 값이었다"는 정보 추가 | val 0.20. 그런데 **제출 안 함** | rolling stats + group stats도 추가해보자 |
| 034~058 통계 feature + weight 압축 | 더 많은 lag/rolling stats, val 방식 개선, weight^0.05 등 대규모 실험 | val 0.59까지 올랐지만 **제출하니 전부 0.0000** | ⚠️ 근본 원인 뒤늦게 발견: group stats(subcat_mean 등)이 원인 |
| 067 group stats 제거 ← 지금 여기 | 분석 결과: subcat_mean = -0.67을 모르는 시리즈에 예측하면 실제값(≈0)보다 더 나쁜 예측이 됨 → ratio > 1 → 0점. sub_01처럼 raw feature만 쓰되 아는 시리즈엔 series_mean/last_y 유지 | 진행 중 | - |

## churn

**한 줄 요약**: 은행 고객이 이탈할지 예측. AUC-ROC 지표 (높을수록 좋음, 최대 1.0).
**핵심 난관**: 상위 팀들이 전부 0.914 근방에 몰려있어서 0.001 개선이 순위를 수십 계단 바꿈.

### 실험 흐름

| Trial | 왜 시도했나 | 결과 | 다음엔 |
|-------|------------|------|--------|
| 001 기본 모델 | 데이터 파악 겸 LightGBM baseline. val-public gap이 얼마나 되는지 확인 | val 0.9161, public 0.9138. gap -0.002로 안정적 | 범주형 변수("인터넷 서비스 있음/없음" 등)를 숫자로 단순 변환했는데, 이탈률 자체로 변환(target encoding)하면 더 직접적인 신호가 될 것 |
| 002~003 feature 설계 | "월 요금이 가입 초반보다 많이 올랐으면 이탈할 것"이라는 가설 → ChargeGap 변수 추가 / CatBoost로 범주형 자동 처리 | val 0.9162~0.9163. 미미한 개선 | 아직 하이퍼파라미터가 기본값 → Optuna로 탐색 |
| 004 하이퍼파라미터 | Optuna 50회 탐색으로 num_leaves, lr, regularization 최적화 | val 0.9166 → **public 0.9139**. 단일 모델 한계 체감 | LightGBM 혼자론 한계. XGBoost/CatBoost와 섞으면 각 모델의 강점을 결합할 수 있음 |
| 005~008 앙상블 | 모델마다 학습하는 패턴이 달라서 평균내면 오차가 줄어듦(앙상블 효과) | val 0.9167, **public 0.9140**. +0.001 개선 | OOF(훈련 데이터를 겹치지 않게 나눠서 예측)로 각 모델 비중을 수학적으로 최적화 |
| 009~013 실패한 시도들 | 외부 Telco 원본 데이터 7천행 추가, "No internet" 값 통합, 10-Fold 등 | 전부 val 하락. 기존 대비 노이즈만 추가한 꼴 | 데이터 분포가 달라서 외부 데이터는 독 |
| 014~017 최적 blend | OOF grid search로 5모델 최적 비중 계산 (XGBoost 0.4 주도) | val 0.9168, **public 0.9140** (동일). 여기가 천장인 것 같음 | multi-seed로 분산 줄이기 시도 |
| 018~024 총동원 | 고객별 groupby 집계 83개 변수, 그룹 내 상대적 위치, XGBoost 7 seeds × 5 fold = 35모델 | val 0.9169(최고) → **public 0.9140 미돌파**. val만 올라가는 과적합 신호 | val 0.914보다 올리는 게 진짜 어렵다. 새로운 feature 아이디어 필요 |
